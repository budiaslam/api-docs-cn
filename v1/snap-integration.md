# Snap 支付网关集成

### 说明

**Snap** 是一种支付方式，可帮助您打造流畅无缝的结账体验。在与 **Snap** 服务交互时，您需要在每个请求中包含特定的请求头（headers）。这些请求头可帮助我们的系统验证并处理您的交易。

### 必需请求头

为确保交易安全，**SNAP API** 采用由印度尼西亚央行制定的安全标准，通过签名与加密机制来保障交易过程的安全性。

我们使用 **非对称加密（Asymmetric Without Get Token）** 类型来生成签名，因此商户无需额外发起 **Get Token** 请求即可创建签名。

该签名会在请求时通过 **X-Signature** 请求头发送。除了 **X-Signature** 之外，请求中还必须包含以下必填请求头：

| Header       | Description                                                                          |
| ------------ | ------------------------------------------------------------------------------------ |
| X-TIMESTAMP  | <p>当前时间戳，采用 <strong>ISO8601</strong> 格式。</p><p>(e.g., 2024-12-19T17:07:05+07:00)</p> |
| X-SIGNATURE  | 商户生成的签名                                                                              |
| X-PARTNER-ID | 提供给商户的客户端密钥                                                                          |
